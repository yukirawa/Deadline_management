# 期限管理アプリ 仕様書

## 1. 目的

学校の**提出物**・**小テスト**・（必要なら）**定期テスト**などの締切を一元管理し、
「近い順に見える」「危険度が一目で分かる」状態を作る。

* 最優先：**締切を落とさない**
* 次点：入力がラク、見るのが速い
* 将来：通知・同期・ウィジェットも可能な設計

## 2. 対象環境

* 開発環境：Windows / VS Code / VS Code（Codex）
* 実行環境：Android（実機 or エミュレータ）
* フレームワーク：Flutter（Dart）

> iOSは将来対応（Macが必要になる可能性が高いので、初期はAndroidで検証）

## 3. 用語

* タスク：提出物・小テストなど、期限を持つ項目
* 締切：提出/実施の期限（日付）
* 危険度：締切までの日数に応じた状態（赤/黄/緑）

## 4. スコープ

### 4.1 MVP（最小完成形）

必ず入れる：

1. タスク追加（科目/種別/内容/締切日）
2. タスク一覧（締切が近い順）
3. 危険度表示（色とラベル）
4. 完了チェック（完了/未完了）
5. ローカル保存（アプリ再起動しても残る）

### 4.2 v0.2（次の一手）

* フィルタ（科目、種別、完了表示切替）
* 検索（内容の部分一致）
* 編集（タスクの更新）
* 削除（確認ダイアログ付き）

### 4.3 v1.0以降（拡張）

* 期限通知（前日/当日など）
* クラウド同期（Firebase等）
* ホーム画面ウィジェット（Android）

## 5. 画面仕様

### 5.1 画面一覧

* A. ホーム（タスク一覧）
* B. 追加/編集（フォーム）
* C. 設定（v0.2以降）

### 5.2 ホーム（タスク一覧）

#### 表示要素

* 上部：アプリ名「締切レーダー」
* 右上：＋（追加）
* （v0.2）検索アイコン / フィルタ

#### 一覧表示（カード or リスト）

各タスク行：

* 危険度バッジ（色：赤/黄/緑/灰）
* 科目（例：数学）
* 種別（提出物/小テスト/定期テスト）
* 内容（例：ワークp.12〜15）
* 締切日（YYYY/MM/DD）
* 残り日数（例：あと2日、今日、期限切れ）
* 完了チェック（チェックボックス）

#### 並び順

* 未完了を上、完了を下
* 未完了の中では締切日が近い順
* 期限切れ（過去日）は最上部に出してもよい（設定で変更可能）

### 5.3 追加/編集フォーム

#### 入力項目

* 科目（必須）

  * 入力方式：ドロップダウン（よく使う科目）＋自由入力
* 種別（必須）

  * 提出物 / 小テスト / 定期テスト（初期は提出物・小テストだけでもOK）
* 内容（必須）

  * 例：プリントNo.3、ワークp.10〜12
* 締切日（必須）

  * 日付ピッカー

#### ボタン

* 保存
* キャンセル

#### バリデーション

* 必須項目が空なら保存不可
* 締切日が未選択なら保存不可

## 6. 危険度ロジック

### 6.1 日数計算

* `daysLeft = dueDate(00:00) - today(00:00)` の日数差
* タイムゾーン：JST（端末のローカル時間）

### 6.2 危険度の定義（初期案）

* **期限切れ**：daysLeft < 0 → 灰（⚠期限切れ）
* **赤**：0〜1日 → 「危険」
* **黄**：2〜3日 → 「注意」
* **緑**：4日以上 → 「余裕」

※境界は設定で変えられる設計にすると強い（v0.2以降）

## 7. データ仕様

### 7.1 Taskモデル

* id: String（UUID）
* subject: String
* type: String（enum相当：assignment/quiz/exam）
* title: String
* dueDate: String（ISO8601推奨：YYYY-MM-DD）
* done: bool
* createdAt: int（epoch ms）
* updatedAt: int（epoch ms）

### 7.2 永続化（MVP）

* 方式：`shared_preferences` にJSON文字列として保存
* キー例：`tasks_v1`
* 保存単位：タスク配列全体

> タスク数が増えたらHive/SQLite移行も可能（v1.0以降）

## 8. 非機能要件

* 起動から一覧表示まで1秒台を目標（体感）
* オフラインで完結
* 端末内データはユーザーが消せる（アプリ削除で消える）

## 9. 例外・エッジケース

* 期限が今日：表示は「今日」
* 期限が過去：期限切れ扱い、上に出すかは仕様で固定
* 同日締切が複数：作成日時で安定ソート
* 入力ミス対策：削除は確認ダイアログ

## 10. 受け入れ基準（MVP）

1. タスクを追加でき、一覧に表示される
2. アプリを終了→再起動してもタスクが残る
3. 締切が近い順に並ぶ
4. 残り日数と危険度（色/ラベル）が正しい
5. 完了チェックを付けると完了扱いになる（下に移動）

## 11. 実装方針（開発手順）

1. Flutterプロジェクト生成
2. Taskモデル＋JSON変換
3. shared_preferencesで読み書き
4. 一覧UI（ListView）
5. 追加UI（フォーム＋DatePicker）
6. 危険度計算とバッジ表示
7. 完了チェック反映

## 12. 将来の設計メモ

* 同期：Firebase（Authなしでも匿名運用は可能）
